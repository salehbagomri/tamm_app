## دورك

أنت مهندس برمجيات خبير متخصص في Flutter وتطوير تطبيقات الجوال. مهمتك هي إنشاء **خطة تنفيذ شاملة ومفصّلة** لتطبيق Flutter اسمه **تمّ**، بحيث يستطيع نموذج ذكاء اصطناعي أقل قدرة تنفيذ كل جزء منها بشكل مستقل دون الحاجة إلى تفكير معماري إضافي.

**مبدأ الخطة:** كل قسم يجب أن يكون واضحاً لدرجة أن النموذج المنفذ لا يحتاج أن يسأل أي سؤال — كل قرار معماري وتقني وتصميمي موثّق مسبقاً.

---

## نبذة عن المشروع

**تمّ** — تطبيق Flutter متكامل لخدمات التكييف والطاقة الشمسية في السوق اليمني.

### الفكرة الجوهرية
منصة تجمع شراء المنتجات + تركيبها + صيانتها في تجربة واحدة سلسة.
الميزة التنافسية: العميل يختار المكيف ويحدد موعد التركيب في **نفس الطلب**.

### المستخدمون الثلاثة
1. **العميل** — يشتري ويطلب الخدمات
2. **المدير** — يستلم الطلبات ويوزعها على الفنيين
3. **الفني** — يستلم مهامه اليومية وينفذها

### الخدمات المقدمة
- شراء مكيفات (أسعار ثابتة)
- تركيب مكيف (اشتراه العميل من أي مكان)
- صيانة وإصلاح وغسيل مكيفات
- تركيب وصيانة منظومات الطاقة الشمسية
- استشارة فنية
- بعض منتجات الطاقة الشمسية بطلب عرض سعر

### القرارات التجارية المعتمدة
- الدفع: **كاش عند الاستلام فقط** (لا دفع إلكتروني في المرحلة الأولى)
- النطاق: مدينة واحدة
- الفنيون: فريق ثابت يعرفهم المدير مسبقاً (ليس نظام توظيف مفتوح)
- التسعير: أسعار ثابتة يضبطها المدير من لوحة التحكم

## ما تحتوي عليه الخطة التفصيلية

اكتب خطة تنفيذ تغطي كل ما يلي بالتفصيل الكامل:

---

### القسم الأول — إعداد المشروع
- أمر إنشاء المشروع بالخيارات الصحيحة
- محتوى `pubspec.yaml` الكامل
- إعداد `AndroidManifest.xml` (الأذونات المطلوبة)
- إعداد `Info.plist` لـ iOS
- إعداد Supabase (متغيرات البيئة بشكل آمن)
- هيكل المجلدات الكامل مع شرح كل ملف

### القسم الثاني — قاعدة البيانات
- SQL كامل لإنشاء كل جدول مع التعليقات العربية
- كل العلاقات والـ Foreign Keys
- كل الـ RLS Policies بالتفصيل
- الـ Functions والـ Triggers المطلوبة (مثل: تحديث حالة الطلب تلقائياً)
- بيانات أولية للاختبار (seed data)

### القسم الثالث — النظام والـ Core
- كود `AppColors` كامل
- كود `AppTextStyles` كامل (كل الستايلات من Harmattan)
- كود `AppTheme` كامل (ThemeData)
- كود `AppRouter` كامل مع كل الراوتات والـ Guards
- كل الـ Shared Widgets مع الكود الكامل لكل واحد

### القسم الرابع — نظام المصادقة (Auth)
- شاشة الترحيب / Splash
- شاشة تسجيل الدخول برقم الجوال + OTP
- منطق تحديد نوع المستخدم (عميل / مدير / فني) بعد الدخول
- تأمين الراوتات حسب الدور
- نموذج `UserProfile` و Provider كاملان

### القسم الخامس — تطبيق العميل

**الشاشة الرئيسية (Home)**
- البانر الترحيبي
- بطاقات الخدمات السريعة (تركيب / صيانة / استشارة)
- قسم المنتجات المميزة
- آخر الطلبات

**المتجر (Store)**
- قائمة المنتجات مع الفلترة حسب الفئة
- بطاقة المنتج (صورة + اسم + سعر + زر إضافة للسلة)
- شاشة تفاصيل المنتج
- سلة المشتريات
- شاشة إتمام الطلب (الاسم + العنوان + الموعد + خيار إضافة تركيب)
- شاشة تأكيد "تمّ طلبك ✓"

**الخدمات (Services)**
- قائمة الخدمات المتاحة
- نموذج طلب الخدمة (نوع الخدمة + العنوان + الموعد + ملاحظات)
- شاشة تأكيد "تمّ الحجز ✓"

**حسابي (Profile)**
- بيانات العميل
- أجهزتي المسجلة (إضافة / تعديل / حذف)
- طلباتي (نشطة + سابقة)
- تفاصيل الطلب مع تتبع الحالة

### القسم السادس — لوحة تحكم المدير

**الرئيسية (Dashboard)**
- إحصائيات اليوم (طلبات جديدة / منجزة / معلقة)
- آخر الطلبات الواردة
- حالة الفنيين (متاح / مشغول)

**إدارة الطلبات**
- قائمة كل الطلبات مع الفلترة والبحث
- شاشة تفاصيل الطلب
- تعيين فني للطلب
- تغيير حالة الطلب
- إصدار فاتورة بسيطة

**إدارة الفنيين**
- قائمة الفنيين مع حالتهم
- إضافة / تعديل فني
- عرض مهام فني معين

**إدارة المنتجات**
- قائمة المنتجات
- إضافة / تعديل / حذف منتج
- رفع صورة المنتج

### القسم السابع — شاشات الفني

**مهامي اليوم**
- قائمة المهام مرتبة بالأولوية
- تفاصيل المهمة (العميل + العنوان + الخدمة + رقم التواصل)
- زر "بدأت" و"أنهيت"
- إمكانية إضافة ملاحظة أو صورة

### القسم الثامن — نظام الإشعارات
- إعداد Supabase Realtime للإشعارات الفورية
- إشعار للمدير عند ورود طلب جديد
- إشعار للعميل عند تغير حالة طلبه
- إشعار للفني عند تعيين مهمة جديدة له

### القسم التاسع — التفاصيل التقنية لكل شاشة
لكل شاشة وثّق:
- اسم الملف ومساره الكامل
- الـ Widget Tree الكامل
- الـ Providers المستخدمة
- الـ Repository calls
- حالات التحميل والخطأ والفراغ
- الـ Navigation (من أين تأتي، وأين تذهب)

### القسم العاشر — الاختبار والتسليم
- checklist ما يجب اختباره قبل الإطلاق
- إعداد بيانات اختبار واقعية
- أوامر البناء للـ APK

## تعليمات صياغة الخطة

1. **اكتب الكود الكامل** — لا تكتب "اكتب كود هنا"، اكتب الكود الفعلي
2. **كل ملف = قسم مستقل** — اذكر المسار الكامل قبل كل كود
3. **لا تختصر** — النموذج المنفذ لن يفكر، فقط ينفذ
4. **اللغة** — كل النصوص في الواجهة عربية، أسماء المتغيرات والكلاسات إنجليزية
5. **الخط Harmattan** — استخدمه في كل نص بدون استثناء
6. **الألوان** — من `AppColors` فقط، لا ألوان مكتوبة مباشرة
7. **اتجاه النص** — `Directionality(textDirection: TextDirection.rtl)`
8. **التصميم** — داكن، احترافي، يعكس هوية تمّ

## الناتج المتوقع من النموذج المتقدم

خطة تنفيذ بـ **١٠ أقسام كاملة** تحتوي على:
- كل أوامر الـ Terminal اللازمة
- كل ملفات SQL لقاعدة البيانات
- كل ملفات Dart بكودها الكامل
- كل شاشة Flutter بـ Widget Tree كاملة
- كل Provider و Repository بكودها الكامل
- ترتيب التنفيذ خطوة بخطوة

**الهدف:** أن يستطيع نموذج أقل قدرة تنفيذ الخطة قطعة قطعة دون أي غموض.
