#### ١ — اختيار الموعد البصري الاحترافي

  اليوم   الأحد   الاثنين   الثلاثاء   الأربعاء   الخميس   الجمعة
   ١٦      ١٧       ١٨         ١٩          ٢٠         ٢١       ٢٢

### المستوى الثاني — تحسين نموذج الطلب

#### ٢ — إعادة تصميم نموذج طلب الخدمة

المشكلة الحالية: النموذج موجود لكن التدفق البصري غير واضح.

النموذج المحسّن — خطوات واضحة ومتسلسلة:

الخطوة ١ — الخدمة المختارة (معروضة للقراءة فقط):
  بطاقة تعرض: اسم الخدمة + السعر + الأيقونة
  لا يستطيع العميل تعديلها — فقط للتأكيد البصري

الخطوة ٢ — الموقع:
  زر "استخدم موقعي الحالي" (GPS)
  أو حقل نص لوصف الموقع
  مطلوب: لا يكمل بدونه

الخطوة ٣ — الموعد:
  منتقي الموعد الاحترافي (كما وُصف في النقطة ١)
  يظهر الموعد المختار بوضوح
  مطلوب: لا يكمل بدونه

الخطوة ٤ — ملاحظات (اختياري):
  حقل نص متعدد الأسطر
  placeholder: "أي تفاصيل إضافية تساعد الفني..."
  اختياري

الخطوة ٥ — ملخص وتأكيد:
  بطاقة ملخص في الأسفل تظهر:
    الخدمة + السعر + الموعد + الموقع
  زر "اطلب الآن" ثابت في الأسفل (sticky)
  زر معطّل حتى تكتمل الحقول الإلزامية
  بعد الضغط: loading ثم شاشة تأكيد "تمّ حجزك ✓"

#### ٣ — شاشة تأكيد الطلب

بعد إرسال الطلب تظهر شاشة كاملة (ليس dialog صغير):

المحتوى:
  أيقونة نجاح كبيرة بلون success
  نص: "تمّ حجزك ✓"
  رقم الطلب
  ملخص: الخدمة + الموعد + الموقع
  نص: "سيتواصل معك الفني قريباً"
  زران:
    "تتبع الطلب" ← يفتح تفاصيل الطلب
    "العودة للرئيسية"

### المستوى الثالث — تفعيل عرض السعر كاملاً

#### ٥ — منطق عرض السعر

هذا المسار الكامل الذي يجب بناؤه:

من جانب العميل:
  العميل يختار "تركيب منظومة شمسية"
    ← يفتح نموذج طلب عرض السعر المخصص:
        وصف الاحتياج (عدد الغرف، الاستهلاك اليومي التقريبي)
        الموقع (إلزامي)
        ملاحظات إضافية
        رقم التواصل (مأخوذ من الحساب تلقائياً — للعرض فقط)
    ← يضغط "اطلب عرض السعر"
    ← شاشة تأكيد: "تمّ إرسال طلبك — سنرسل لك العرض قريباً"

من جانب المدير (في لوحة التحكم):
  يظهر الطلب بنوع "طلب عرض سعر" مميز بلون مختلف
  المدير يفتح الطلب ويرى تفاصيل الاحتياج
  المدير يدخل:
    السعر الإجمالي
    تفاصيل العرض (ماذا يشمل)
    مدة التنفيذ التقديرية
    ملاحظات إضافية
  يضغط "أرسل العرض للعميل"
  ← إشعار فوري للعميل

من جانب العميل بعد استلام العرض:
  إشعار: "تمّ إرسال عرض السعر لطلبك"
  العميل يفتح الطلب ويرى:
    السعر الإجمالي بارز
    تفاصيل ما يشمله العرض
    مدة التنفيذ
    ملاحظات المدير
  زران واضحان:
    "قبول العرض ✓" بلون success
    "رفض العرض" بلون error
  إذا قبل: يتحول لطلب عادي ويبدأ مسار التنفيذ الطبيعي
  إذا رفض: يُغلق الطلب مع إمكانية إضافة سبب الرفض

### المستوى الرابع — تحسينات إضافية

#### ٦ — شاشة تفاصيل الخدمة

حالياً العميل يضغط على الخدمة مباشرة ويفتح النموذج.
المطلوب: شاشة وسيطة تعرض تفاصيل الخدمة قبل الطلب.

المحتوى:
  أيقونة الخدمة كبيرة
  اسم الخدمة
  وصف تفصيلي (ماذا تشمل الخدمة)
  السعر بارز أو "عرض سعر"
  قائمة "ماذا يشمل":
    مثال لتركيب مكيف سبليت:
    ✓ توريد جميع مواد التمديد
    ✓ تركيب الوحدة الداخلية والخارجية
    ✓ اختبار التشغيل
    ✓ ضمان سنة على التركيب
  زر "احجز الآن" ثابت في الأسفل

---

#### ٧ — حالات الشاشة الثلاث

كل حالة يجب معالجتها في شاشة الخدمات:

loading: Shimmer يحاكي شكل بطاقات الخدمات
error: رسالة "تعذّر تحميل الخدمات" + زر "حاول مجدداً"
empty: رسالة "لا توجد خدمات متاحة حالياً" (نادر لكن ضروري)

## تعديلات قاعدة البيانات المطلوبة

### تعديل جدول service_types
إضافة أعمدة:
  category (text): مكيف أو طاقة_شمسية أو استشارة
  is_quote_based (boolean): هل هي خدمة عرض سعر؟
  includes (text[]): قائمة ما تشمله الخدمة (للعرض في التفاصيل)
  estimated_duration (text): مدة تنفيذ تقديرية مثل "٢-٤ ساعات"

### تعديل جدول orders
إضافة أعمدة:
  scheduled_period (text): صباحاً أو ظهراً أو مساءً
  scheduled_hour (text): الساعة المحددة (اختياري)
  quote_price (numeric): السعر الذي أدخله المدير لعرض السعر
  quote_details (text): تفاصيل عرض السعر
  quote_duration (text): مدة التنفيذ التقديرية
  quote_status (text): pending أو accepted أو rejected
  quote_sent_at (timestamptz): وقت إرسال العرض
  quote_responded_at (timestamptz): وقت رد العميل

### RLS الإضافية
  العميل يرى طلباته فقط
  المدير يرى كل الطلبات
  المدير فقط يعدّل quote_price و quote_details و quote_status
  العميل فقط يعدّل quote_status عند القبول أو الرفض
